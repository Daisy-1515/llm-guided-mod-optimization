## Phase⑥ Step 1：推进力功率与飞行能耗模块

接上午 Phase⑤ 收口后，启动 Phase⑥ BCD 循环的最底层依赖：UAV 推进力模型。

### 实现内容

新建 `edge_uav/model/propulsion.py`（130 行），实现三个公开函数：

| 函数 | 公式 | 作用 |
|------|------|------|
| `propulsion_power(v_sq, ...)` | Eq.(18) | 给定速度平方，返回推进功率 (W) |
| `flight_energy_per_slot(q_j, delta, ...)` | Eq.(3')+Eq.(17) | 单 UAV 全时隙飞行能耗 |
| `total_flight_energy(q, delta, ...)` | 聚合 | 所有 UAV 的总飞行能耗 |

### 关键设计决策

1. **接收 `v_sq`（速度平方）**而非 `v` — 轨迹优化中常用 ‖Δq‖² 避免开方
2. **induced term 使用有理化形式** — `η₃ / (sqrt(η₃ + v⁴/4) + v²/2)` 避免大速度时的浮点精度丢失（Codex review 建议采纳）
3. **最后时隙按悬停处理** — 无 q[t+1] 时 v=0
4. **类型别名与 precompute.py 对齐** — `Trajectory2D = dict[int, dict[int, tuple]]` 为 fleet 级别
5. **keyword-only 参数** — 不依赖 config 对象，便于测试和复用

### U 形功率曲线

测试中发现旋翼功率曲线是 U 形的（悬停功率 ~111W 高于中速飞行功率），这是物理正确的行为：
- 悬停时 induced power 占主导
- 中速时 induced power 快速衰减，而 blade profile + parasite drag 增长较慢
- 高速时 parasite drag (v³) 项占主导

初始的单调递增假设有误，已修正测试为 U 形曲线验证。

#### 关键创新：对偶二分法 vs 等比缩放

原计划文档提议「水填充等比缩放」，但在异构任务（不同 τ_i）下**违反 KKT 条件**。

**改进方案：对偶二分法**

给定卸载决策和轨迹，频率优化的 KKT 一阶条件：

```
∂L/∂f_ji = -α·F_i/(f²·τ_i) + 2·γ_w·γ_j·f·F_i/E_max + λ = 0
```

令 a_i = α·F_i/τ_i, b_i = γ_w·γ_j·F_i/E_max，得：

```
2b_i·f³ + λ·f² - a_i = 0  （每任务的三次方程）
```

**求解步骤：**

1. **无约束 KKT**（λ=0）：f* = (a_i/(2b_i))^(1/3)
2. **容量约束触发**（Σf* > f_max_j）：对偶二分法在 λ ∈ [0, λ_max] 上搜索，使 Σf_i(λ) = f_max_j
3. **三次方程求解**：二分法在 [eps_freq, f_max_j] 上逆解正根（60 次迭代）

**优势：**
- ✓ 异构任务下精确解（vs. 等比缩放的近似）
- ✓ BCD 每步成本单调不递增（收敛性保证）
- ✓ 额外仅需 ~20 行代码
