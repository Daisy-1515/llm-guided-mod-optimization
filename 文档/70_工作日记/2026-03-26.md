## 📝 技术细节

### SOCP 改写的数学等价性

**通信延迟约束**：
```
原：2√(H²+||Δq||²) / rate ≤ τ
改：||[H, Δq]||₂ ≤ s ∧ 2s ≤ τ·rate
```
完全等价，无近似误差。

**速度链接约束**：
```
原：speed_sq == ||Δq||² / δ²
改：||Δq||² ≤ δ² · speed_sq
```
改为不等式，允许速度≥最小值，但由于目标是最小能耗，优化器自动选择最小。

### 两个失败的测试（已知问题）

1. **T1：数值精度**（test_stationary_endpoint_returns_hover_path）
   - 预期：q[0][1] = 0.0
   - 实际：q[0][1] = 1.66e-5
   - 原因：浮点精度 + 不等式约束的松弛
   - 建议：调整容差或增加 SCA 迭代

2. **T9：预算检查**（test_negative_communication_budget_raises）
   - 预期：抛出"Infeasible communication budget"
   - 实际：正常运行，不抛错
   - 原因：负数预算检查逻辑未实现
   - 建议：在 solve_trajectory_sca() 中添加预验证
