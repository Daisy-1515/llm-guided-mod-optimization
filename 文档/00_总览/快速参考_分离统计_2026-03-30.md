# 快速参考：分离统计指标

## 🎯 核心问题解答

**Q: offload_feasible_ratio 从 50% 跌到 0.7%，这是为什么？**

A: 由于混合统计，现在可以分离看：
- `assigned_feasible_ratio`: 已分配对的真实可行率（排除 f=0 的对）
- `offload_feasible_ratio`: 包括 f=0 对的混合可行率

---

## 📊 新增指标一览

| 指标 | 值范围 | 含义 | 如何用 |
|------|--------|------|--------|
| `assigned_feasible_ratio` | 0.0-1.0 或 None | 已分配对的可行率 | None = 无分配；0.0 = 全不可行；> 0.5 = 压力适中 |
| `assigned_pair_ratio` | 0.0-1.0 | 系统分配率 | > 0.8 充分利用，< 0.3 潜力大 |
| `assigned_pairs` | 整数 | 被分配的对数 | 系统实际决策的规模 |
| `unassigned_pairs` | 整数 | 未分配对数 | 受约束限制的对数 |

**关键点**：`assigned_feasible_ratio` 可能为 `None`，表示无已分配对（不是可行率为 0）

### 查看诊断数据（Python）
```python
import json

run_id = "20260330_131259"  # 替换成最新运行号
with open(f"discussion/{run_id}/population_result_0.json") as f:
    data = json.load(f)[0]  # 取第一个个体

# 提取分离统计（正确的路径）
diag = data["simulation_steps"]["0"]["final_precompute_diagnostics"]

print(f"混合可行率: {diag['offload_feasible_ratio']:.1%}")

# 注意：assigned_feasible_ratio 可能为 None
assigned = diag.get('assigned_feasible_ratio')
if assigned is None:
    print("已分配可行率: N/A (无已分配对)")
else:
    print(f"已分配可行率: {assigned:.1%}")

print(f"分配率: {diag['assigned_pair_ratio']:.1%}")
```

### 解读结果

```
Case 1: assigned_feasible_ratio = 50%, offload_feasible_ratio = 1%
解读: 已分配对的压力是 50% 可行
     总体看起来 1% 是因为大量未分配对（f=0）拉低了
     → 这不是真的 deadline 压力，而是统计混淆
     ✓ 设计权衡是合理的

Case 2: assigned_feasible_ratio = None （assigned_pairs = 0）
解读: BCD 优化没有给任何对分配频率
     → 可能约束过紧，或目标函数不可行
     → 检查 N_max / 其他约束配置

Case 3: assigned_feasible_ratio = 15%, offload_feasible_ratio = 2%
解读: 即使只看已分配对，可行率也很低（15%）
     → 这是真实的 deadline 压力
     → 需要调整 alpha / gamma_w 等参数

Case 4: assigned_pair_ratio = 30%
解读: 系统只分配了 30% 的潜在对
     → 约束可能过紧，或有优化空间
     → 但不要直接增大频率，应从约束或目标权重入手
```

## 🔧 参数调整建议

**首先检查**：assigned_pairs > 0？

| assigned_feasible_ratio | 建议 |
|---------|------|
| None（assigned_pairs=0） | 🔴 无分配，检查约束或目标函数 |
| > 70% | ✓ 压力合理，继续当前方案 |
| 50-70% | ⚠ 中等压力，可尝试微调权重 |
| 30-50% | ⚠ 压力较大，建议调整 alpha/gamma_w/lambda_w |
| < 30% | 🔴 压力很大，需要显著调整或重新设计目标函数 |

**独立指标**：
- 若 `assigned_pair_ratio < 0.3`：系统分配率低，检查 N_max 或其他约束是否过紧

A: 下一次运行任何实验就会自动生成。

**Q: 如果 `assigned_feasible_ratio` 不存在怎么办？**
A: 说明运行的是旧代码。请确保已更新 precompute.py 和 bcd_loop.py。

## 📈 分析模板

复制以下代码分析一个运行：

```python
import json
import pandas as pd
from pathlib import Path

def analyze_run(run_id):
    results = []
    for f in sorted(Path(f"discussion/{run_id}").glob("population_result_*.json")):
        with open(f) as fp:
            data = json.load(fp)[0]
            diag = data["simulation_steps"]["0"]["final_precompute_diagnostics"]
            results.append({
                "id": int(f.stem.split("_")[-1]),
                "mixed": diag.get("offload_feasible_ratio", 0),
                "assigned": diag.get("assigned_feasible_ratio"),  # 可能为 None
                "alloc_rate": diag.get("assigned_pair_ratio", 0),
                "score": data.get("evaluation_score", float('nan')),
            })

    df = pd.DataFrame(results)
    print(df)

    # 注意：assigned 可能为 None，需要过滤后再计算平均
    assigned_values = [x for x in df['assigned'] if x is not None]
    if assigned_values:
        print(f"\n统计:")
        print(f"  平均已分配可行率: {pd.Series(assigned_values).mean():.1%}")
    print(f"  平均分配率: {df['alloc_rate'].mean():.1%}")
    return df

# 使用
# df = analyze_run("20260330_131259")
```
