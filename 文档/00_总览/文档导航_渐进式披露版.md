## 7. 今后写新文档时，如何符合渐进式披露

建议所有新文档都遵守下面 6 条规则：

1. 开头先写“这份文档回答什么问题”，不要直接进入细节。
2. 前 20 行必须让读者判断“我该不该继续读”。
3. 一个文档只解决一个主问题，不把总览、方案、排查记录混在一起。
4. 在文档顶部标记类型：`总览 / 设计 / 实现 / 测试 / 审查 / 论文 / 日记 / 历史`。
5. 在文档顶部标记状态：`active / reference / archive`。
6. 总览文档只放入口和决策，不堆公式；公式和实现细节下沉到专题文档。

推荐模板：

```md
# 标题

> 类型：设计
> 状态：active
> 目标读者：准备修改 xxx 模块的人
> 前置阅读：README、文档导航

## 1. 这份文档回答什么问题

## 2. 先给结论

## 3. 再讲方案

## 4. 最后给细节 / 边界 / 风险

## 5. 相关文档
```

1. 只重组文档树，不改代码目录结构。
