# 图片参数与项目参数映射表

本文档用于对照用户提供的参数表图片与当前项目 `Edge UAV` 分支中的实际参数配置和实现方式。

适用范围：
- 配置文件：`config/setting.cfg`
- 配置加载：`config/config.py`
- 场景生成：`edge_uav/scenario_generator.py`
- 预计算与通信模型：`edge_uav/model/precompute.py`
- 启动脚本运行时覆盖：`testEdgeUav.py`

说明：
- “图片参数”指用户在会话中提供的参数表图片。
- “项目当前值”优先指仓库当前代码路径下的默认配置值。
- 如启动脚本会在运行时覆盖配置值，单独在“运行时覆盖”列和备注中说明。
- 若图片中的参数在项目中没有显式实现，会标为“未显式建模”或“模型简化替代”。

---

## 1. 总体结论

当前项目不是对图片参数表的逐项严格复现，而是采用了更简化的建模方案：

- 信道模型采用 `LoS 主导 + 自由空间路径损耗`，即 `g = rho_0 / d^2`。
- 没有显式实现图片中的 LoS/NLoS 概率模型、Nakagami 衰落、阴影衰落、地面 MBS 参数等。
- 已实现的核心参数主要集中在：
  - 通信带宽与发射功率
  - 噪声功率与参考信道增益
  - UAV 推进模型参数
  - 任务数据量、计算量、截止时间
  - UAV 能量预算与 CPU 频率
- 当前启动脚本 `testEdgeUav.py` 会对任务的 `tau` 和 `f_local` 做二次覆盖，这会导致“运行时实际值”与配置文件值不同。

## 3. 已实现但命名或建模方式不同的参数

| 图片符号 | 图片含义 | 项目对应 | 当前实现方式 | 说明 |
|---|---|---|---|---|
| `\mu_i^t` | 任务计算强度 | `F_i` | 在单时隙完成假设下，计算量直接用 `F_i` 表示 | 项目没有单独再采样“每时隙计算强度” |
| `P_j` | UAV 发射功率 | `P_j` | 固定常数 | 图片写成可能带时隙上标，项目当前按常数处理 |

## 6. 运行时覆盖项

当前 `Edge UAV` 启动脚本 `testEdgeUav.py` 在生成完场景后，会统一覆写所有任务的两个字段：

| 字段 | 配置值 | 运行时值 | 影响 |
|---|---:|---:|---|
| `task.tau` | 从 `[0.5, 2.0] s` 均匀采样 | `200.0 s` | 大幅放宽 deadline，和图片参数表不再同一量级 |
| `task.f_local` | `1e9 Hz` | `1e6 Hz` | 大幅压低本地计算能力，和图片参数表不再同一量级 |

这意味着：

- 如果你看的是 `config/setting.cfg`，会得到一套参数。
- 如果你实际执行 `uv run python scripts/testEdgeUav.py`，任务 deadline 和本地算力会变成另一套值。
- 所以做论文参数对齐、图表复现实验或和图片逐项比对时，不能只看配置文件，必须同时看启动脚本。

## 7. 最关键的差异项

如果目标是让项目参数更接近图片参数表，当前最值得优先检查的是以下几项：

1. `N_0`
   当前配置 `1e-10 W` 与图片中的 `-98 dBm` 不一致，而且代码注释里把它写成“约 -100 dBm”，这一点本身就值得复查。

2. `B_up` / `B_down`
   当前只有 `1 MHz`，明显低于图片里的 `10 MHz` 或 `20 MHz`。

3. `E_max`
   当前 UAV 能量预算只有 `5 kJ`，明显低于图片里的 `360 kJ`。

4. `f_max`
   当前 UAV 最大频率为 `5 GHz`，低于图片中的 UAV/MEC 计算资源范围。

5. `d_U_safe`
   当前安全距离 `50 m`，而图片是 `10 m`。

6. `tau` 和 `f_local` 的运行时覆写
   启动脚本把这两项改到了和图片参数表完全不同的量级，会直接影响实验可比性。
