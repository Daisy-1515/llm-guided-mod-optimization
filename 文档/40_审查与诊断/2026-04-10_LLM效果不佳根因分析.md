# LLM 效果不佳根因分析（2026-04-10）

> **问题**：当前实验中 A 组（LLM+HS）相对 D1（固定默认目标）改善仅 2.22%，且 HS 种群从 Gen2 起完全停止进化。
>
> **目标**：识别根本原因，明确哪些是可修复的代码/参数问题，哪些是方法结构性局限。

---

## 2. 根因一：HS 种群多样性崩溃（最直接原因）

### 2.1 现象

- Gen0：10 个个体，评估分数分布在 74.30~116.76（有多样性）
- Gen1+：所有 10 个个体得分收敛至同一数值（74.48~74.98 极小区间）
- 50 次评估中只有 eval=1→2 有改善，此后完全平坦

### 2.2 代码根因（`heuristics/hsPopulation.py`）

**精英压力过强（line 189-201）**：

```python
def generate_new_harmony(self, pop, force_elite=False):
    if force_elite:  # 每代强制至少 1 个精英变异
        ind = pop[0]  # 始终从全局最优变异
        parent = self.shrink_token_size(ind)  # 仅保留目标代码，丢弃推理上下文
        way.append(random.choice([WAY_PITCH, WAY_CROSS]))
        return p[0], way[0], parent_snapshot
```

**选择范围过窄**：

```python
rd = max(int(self.popsize / 2), 0)  # 只从前 50% 个体中选择
idx = random.randint(0, rd - 1)     # popSize=10 时 rd=5，只有前 5 个个体
```

**随机新生成概率过低**：

```python
if random.random() >= self.HMCR:  # HMCR=0.9 → 只有 10% 概率全新生成
    way.append(WAY_RANDOM)
```

**数学分析**：

```
Gen 0: 10 个随机初始化个体（多样）
Gen 1: 1 个精英 + 9 个来自前 5 个个体的变异 → 实际源头 ~5 个
Gen 2: 递推 → 实际源头 ~3 个
Gen 3: → ~2 个
Gen 4: → ~1 个（完全收敛）
```

5 代 × popSize=10 的配置在第 3-4 代时种群已完全失去多样性。


**可行修复**：
- `popSize` 扩大至 20-30
- `HMCR` 降至 0.7（增加随机新生成概率至 30%）
- 启用 `hsDiversitySorting` 防止同分个体聚集
- 去掉强制精英（或改为概率 20%）

## 3. 根因二：BCD 结构性鞍点（方法局限）

### 3.1 现象

BCD 最大迭代 5 次，实际收敛 2-3 轮：

```
BCD iter 1: L1 cost 161 → L2 调整轨迹
BCD iter 2: L1 cost 115（下降 28%）→ L1 决策锁定，L2 再次调整
BCD iter 3: L1 cost 115（不变）→ convergence_flag=True, break
cost_history 末尾两值相差 < eps_bcd=1e-3 → 立即停止
```

### 3.2 根本机制

坐标下降（BCD）的局部最优性质：
- L1 求解 MIP（混合整数规划），整数决策变量（x_local, x_offload）一旦确定，在给定 L2 反馈下不再翻转
- L2 的轨迹优化在固定 L1 决策下也难以引起大幅变化
- 结果：BCD 快速落入鞍点，2 轮即无改善

**HS 的价值本在于**：引导 BCD 落入不同的鞍点（不同目标函数 → 不同 L1 决策 → 不同 BCD 路径）。但当 HS 种群多样性崩溃后，这个优势也消失了。

### 3.3 不可简单修复

BCD 鞍点是坐标下降 + MIP 离散性的结构性局限，不能通过改参数解决。需要考虑：
- 增加 BCD 重启机制（从不同初始点出发）
- 在 L1 中引入随机扰动打破鞍点

## 4. 根因三：通信时延 << 飞行能耗（轨迹无牵引力）

### 4.1 数值失衡

当前配置（seed=42，20 任务/3 UAV）：

| 成本项 | 典型量级 | 权重 | 加权后 |
|--------|---------|------|--------|
| 时延（alpha × Σ D/tau） | ~0.3 | 350 | ~105 |
| 计算能耗（gamma_w × Σ E_comp/E_max） | ~0.01 J/J | 0.01 | ~0.0001 |
| **飞行能耗（lambda_w × E_fly）** | **~200 J** | **1.0** | **~200** |

**飞行能耗远大于时延项**，导致 L2b（轨迹优化）的主要目标是节省飞行能耗（走直线），而非靠近卸载任务。

### 4.2 SINR 引入后问题加剧

SINR 模型引入后卸载时延增大 677%，但：
- D_hat_offload 从 0.031 增至 0.302
- 仍然 << 飞行能耗 200J 的量级
- LLM 生成的目标函数（如 `(D/tau)²`）放大时延项，但在评估时仍被飞行能耗淹没

### 4.3 与历史成功实验的差异

2026-04-02 的 89.7% 改善实验参数（`20260402_214943`）：
- `alpha=1, gamma_w=1, lambda_w=1`（三项权重均等）
- `B_up=4e7 Hz`（带宽更大，通信时延更短）
- `tau=0.5s`（截止期更紧，时延惩罚更大）
- 结果：通信时延与飞行能耗量级相当，LLM 目标有牵引力

当前实验：
- `alpha=350`（时延项放大 350 倍，但飞行能耗绝对值更大）
- `B_up=1e7`（SINR 后带宽更小，卸载时延更大）
- 计算后：`alpha × delay_term ≈ 105`，`lambda_w × fly_energy ≈ 200`

**关键矛盾**：alpha=350 是为了在评估时强调时延，但 L2b 轨迹优化仍以飞行能耗为主导，两层目标不对齐。

## 5. 根因四：评估指标与 D1 目标等价

### 5.1 机制

当前 `evaluate_solution` 的评估函数（`edge_uav/model/evaluator.py`）：

```
score = (1/N_act) × alpha × Σ(delay_normalized × x)
      + (1/N_act) × gamma_w × Σ(E_comp/E_max × x)
      + (1/N_fly) × lambda_w × E_fly
```

D1 的默认目标函数（`offloading.py`）：

```
obj = alpha × Σ(D/tau × x) + gamma_w × Σ(E_comp/E_max × x)
```

两者高度相关。D1 在直接最小化评估函数的一部分（时延 + 计算能耗），而 LLM 需要通过"间接代理目标"找到 D1 无法找到的更好解。

### 5.2 LLM 理论上的优势机制

LLM 的非线性目标（如 `(D/tau)²`）在 BCD 中放大轨迹反馈效果：
- UAV 靠近任务 → `D_hat_offload` 下降 → `(D/tau)²` 急剧下降 → L1 立即卸载该任务
- D1 线性目标对轨迹反馈不敏感

但当前 BCD 2 轮即收敛 + 种群多样性崩溃，这个优势无法发挥。

## 6. 根因五：SINR 引入后数值不稳定

### 6.1 直接影响

- 上行速率从 11.18 Mbps 降至 1.16 Mbps（-89.6%）
- 卸载时延增大 677%
- 部分任务在 `tau=1.0s` 内卸载不可行（需外点罚函数容错）

### 6.2 间接影响

SCA 通信约束线性化下界 `r_up_lb` 在 SINR 下更小（更悲观），导致：
- 硬约束版 SCA 在第一轮迭代即 infeasible
- 已用外点罚函数缓解，但 SCA 数值收敛质量下降
- A 组 feasible_rate 从 100% 降至 84%

## 7. 根因层次与可行性

| 根因 | 类型 | 修复难度 | 预期收益 |
|------|------|---------|---------|
| **HS 种群多样性崩溃** | 可修复（参数 + 代码） | 低（改 popSize/HMCR/启用 Diversity Sorting） | 高（让 HS 真正搜索） |
| **BCD 鞍点** | 结构性局限 | 高（需重启机制） | 中 |
| **飞行能耗 >> 通信时延** | 可修复（参数平衡） | 低（调 lambda_w 或归一化） | 中（取决于参数调到什么程度） |
| **评估-优化目标等价** | 设计问题 | 中（引入非线性评估指标） | 中 |
| **SINR 数值不稳定** | 可缓解（已部分修复） | 已做（外点罚函数） | 已改善 |

---

## 8. 修复优先级建议

### 立即可做（不改模型结构）

1. **扩大种群**：`popSize=20-30`，`HMCR=0.7`，`iteration=10`
2. **启用 hsDiversitySorting**：`hsFrame.py:37` 替换 `hsSorting()` 为 `hsDiversitySorting()`
3. **参数平衡**：调低 `lambda_w` 或加归一化因子，使飞行能耗与时延项量级相当

### 需要设计决策

4. **评估指标引入非线性**：指数截止期惩罚 `exp(max(0, D/tau-1))-1` → 让 D1 无法直接最小化评估函数
5. **BCD 重启机制**：在 HS 每代间重置 BCD 起点
