# 实验操作手册（分机执行版）

## 1. 目的

当前本机性能不足，建议把**彼此独立**的实验拆到不同电脑执行。本文档只记录：

- 现在还需要做的实验
- 每个实验的**具体命令**
- 输出目录规范
- 如何判断实验是否真正完成

> 约定：所有命令都在仓库根目录执行，且必须使用 `uv run python`。

---

## 2. 完成判据（必须看）

一次实验只有在输出目录里同时出现以下文件，才算**有效完成**：

- `manifest.json`
- `comparison_summary.json`
- `<group>/summary.json`
- `<group>/run_seed_42.json`

如果只有 `manifest.json`，说明这次运行**中断 / 死机 / 未完成**，不能采用。

---

## 3. 当前统一参数基线

除非命令中显式覆盖，否则默认基线来自 `config/setting.cfg`。当前建议统一按下面的参数执行：

- `scenario_seed = 42`
- `use_bcd_loop = true`
- `llm_layer = L1`
- `hs popSize = 3`
- `hs iteration = 3`

为了减少不同电脑之间的配置漂移，**优先使用命令行覆盖关键参数**，不要只依赖本地 `setting.cfg`。

## 6. 推荐的分机拆分方式

### 方案 A：两台机器

- **机器 1**：整理无人机数消融汇总
  - 统一 `u=3~9` 口径并生成汇总表
- **机器 2**：跑任务数补点
  - `tasks=25`

### 方案 B：三台机器

- **机器 1**：整理 `u=3~9` 无人机消融结果
- **机器 2**：`tasks=25`
- **机器 3**：如需补历史 pure-LLM 旧批次，则单独重跑对应 B 组点位，不要复用旧 manifest-only 目录

### 方案 C：更多机器并行

如果机器更多，原则是：

- **一台机器同时只跑 1 条 A 组实验**
- 不要在同一台机器上并发多条 `A` 组命令
- 不同机器必须使用**不同的输出目录**

---

## 7. 执行前检查清单

每台机器执行前先做：

```bash
uv sync
```

然后确认仓库根目录正确，再执行实验命令。

如果担心本地配置被改过，优先使用本文档里的**全参数命令**，不要自己手动改 `setting.cfg` 后再跑。

---

## 8. 执行后如何验收

假设你跑的是 `u=8`，验收步骤：

```bash
ls -R "最后实验/横坐标无人机 消融实验/u8"
```

必须看到类似：

```text
最后实验/横坐标无人机 消融实验/u8/<timestamp>/
  manifest.json
  comparison_summary.json
  A/
    summary.json
    run_seed_42.json
```

如果只有：

```text
manifest.json
```

则说明这次结果无效，需要重跑。

## 10. 最稳妥的执行原则

1. **命令里写全关键参数**，不要只依赖本地配置
2. **同一实验不要多个机器写同一个输出目录**
3. **只认 `summary.json + run_seed_42.json`，不认单独的 manifest**
4. **死机/断电后的目录默认视为无效，直接重跑**
