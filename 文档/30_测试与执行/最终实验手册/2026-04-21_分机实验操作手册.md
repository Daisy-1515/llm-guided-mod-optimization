# 分机实验操作手册（2026-04-21）

## 1. 目的

由于单机性能不足，将实验拆分到多台机器并行执行。本手册给出：

- 必做实验清单
- 每个实验的直接执行命令
- 推荐分机分工
- 结果目录规范
- 跑完后的分析与回传方式

---

## 2. 环境准备

在每台机器的项目根目录执行：

```bash
uv sync
```

检查 API / 环境：

```bash
uv run python scripts/check_llm_api.py
```

说明：

- 跑 `D1` 基线时，对 LLM API 依赖较弱。
- 跑 `A` 组时，建议先确认 API 正常。
- 所有 Python 命令统一使用 `uv run python`。

---

## 3. 当前推荐统一基线

除非命令里显式覆盖，否则默认参数来自 `config/setting.cfg`。

为了保证跨机器可比性，建议实验统一使用以下命令行覆盖值：

- `numTasks = 20`
- `numUAVs = 3`
- `tau = 1.0`
- `B_up = 1e7`
- `B_down = 1e7`
- `HS popSize = 20`
- `HS iteration = 8`
- `BCD = 开启`

这组参数与近期记录中的正式验证方向一致，适合作为主对比基线。

---

## 4. 结果目录规范

每台机器都必须把结果输出到独立目录，避免覆盖：

```bash
discussion/<机器名>_<实验名>
```

示例：

- `discussion/pc1_a_vs_d1_pop20`
- `discussion/laptop_tau_sweep`
- `discussion/server_bandwidth_sweep`

---

## 5. 必做实验总表

优先级建议：

1. `A vs D1` 公平对比
2. `popSize=20` 完整实验
3. `tau sweep`
4. `bandwidth sweep`
5. UE sweep 补点

如果机器很弱，优先保证：

- `A vs D1`
- `D1` 基线
- 小规模 `bandwidth sweep`

---

# 6. 实验一：A vs D1 公平对比（最高优先级）

## 6.1 目标

验证统一参数下，`A`（LLM+HS）相对 `D1`（默认目标）的稳定收益。

## 6.2 命令

```bash
uv run python scripts/run_all_experiments.py \
  --groups A D1 \
  --seeds 42 43 44 \
  --num-tasks 20 \
  --num-uavs 3 \
  --tau 1.0 \
  --b-up 1e7 \
  --b-down 1e7 \
  --hs-pop-size 20 \
  --hs-iterations 8 \
  --output-root discussion/pc1_a_vs_d1_pop20
```

## 6.3 跑完后分析

```bash
uv run python scripts/analyze_results.py --experiment-dir discussion/pc1_a_vs_d1_pop20
```

如果需要补图：

```bash
uv run python scripts/plot_optimization_curves.py \
  --mode ue-sweep-by-group \
  --metric best_cost \
  --num-uavs 3 \
  --exp-dir discussion/pc1_a_vs_d1_pop20 \
  --output-dir discussion/plots
```

# 13. 推荐分机方案

## 方案 A：3 台机器

- 机器 1：`A vs D1` 主实验
- 机器 2：`tau sweep`
- 机器 3：`bandwidth sweep`

## 方案 B：5 台机器

- 机器 1：`A` 组全部 seeds
- 机器 2：`D1` 组全部 seeds
- 机器 3：`tau sweep`
- 机器 4：`bandwidth sweep`（低带宽）
- 机器 5：`bandwidth sweep`（中/高带宽）+ UE sweep 补点

## 方案 C：很多弱机器

把实验拆成最小粒度：

- 一台机器只跑一个 seed
- 或一台机器只跑一个 `numTasks`
- 或一台机器只跑一个带宽值

这样最稳，失败后也更容易补跑。

1. 结果目录

4. 如果失败，提供最后 50 行日志

```text
机器名:

输出目录:

```
