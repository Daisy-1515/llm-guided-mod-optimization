## LLM API 配置

### 配置两个文件

#### 1. `config/setting.cfg` — 平台和模型选择

```ini
[llmSettings]
platform = HuggingFace      # Factory 路由
model = qwen3.5-plus        # 具体模型 ID
```

**支持的模型**：
- `qwen3.5-plus` — 推荐，快速稳定
- `deepseek-chat` — 有思考能力的版本
- `gpt-4o` — OpenAI 模型
- `glm-5` — 智谱清言（注意：响应时间较长）

#### 2. `config/env/.env` — API 凭证（NOT git tracked）

```env
HUGGINGFACE_ENDPOINT = "https://your-proxy-domain/v1"
HUGGINGFACEHUB_API_TOKEN = "sk-your-api-key"
```

**注意**：
- `.env` 文件 NOT 检入 Git（已在 `.gitignore`）
- 每个人需要自己的 API 凭证
- 端点自动规范化：`/v1` → `/v1/chat/completions`

### API 路由

项目使用 Factory 模式，统一处理多个 LLM 提供商：

```python
# llmAPI/llmInterface.py
class LlmInterface:
    @staticmethod
    def get_llm_api():
        if platform == "HuggingFace":
            return HuggingFaceAPI(...)  # OpenAI 兼容格式
        # ... 其他平台
```

**优势**：
- 任何 OpenAI 兼容的代理都能用（CloseAI、vLLM 等）
- 响应解析支持 3 格式（`<think>` tags、JSON、纯文本）
- 自动 endpoint 规范化

### 当前部署

- **代理**: CloseAI (`api.openai-proxy.org`)
- **模型**: qwen3.5-plus
- **特性**: 快速（3-15s）、稳定、推荐复杂提示

## Harmony Search 参数

### 配置位置

`config/setting.cfg` 的 `[hsSettings]` 部分：

```ini
[hsSettings]
popSize = 5        # 种群大小（5 个个体）
iteration = 10     # 进化代数
```

### 参数说明

| 参数 | 范围 | 推荐 | 说明 |
|------|------|------|------|
| `popSize` | 2-20 | 5-10 | 更大 = 更多样化但更慢 |
| `iteration` | 1-100 | 10-50 | 更多 = 更优但更慢 |

### 调整建议

**快速试验**：
```ini
popSize = 3
iteration = 5
```

**生产运行**：
```ini
popSize = 10
iteration = 50
```

**参数扫描**：
```bash
HS_POP_SIZE=10 HS_ITERATION=20 uv run python scripts/testEdgeUav.py
```

## 常见配置任务

### 任务 1：切换 LLM 模型

**目标**：从 qwen3.5-plus 换到 deepseek-chat

**步骤**：
1. 编辑 `config/setting.cfg`：
   ```ini
   [llmSettings]
   model = deepseek-chat
   ```

2. 检查 API 连接：
   ```bash
   uv run python scripts/check_llm_api.py
   ```

3. 运行测试：
   ```bash
   uv run python scripts/check_llm_api.py
   ```

### 任务 2：修改 HS 参数

**目标**：运行更多代数的进化（迭代 30 次）

**步骤**：
```bash
HS_POP_SIZE=5 HS_ITERATION=30 uv run python scripts/testEdgeUav.py
```

或编辑 `config/setting.cfg`：
```ini
[hsSettings]
iteration = 30
```

### 任务 3：调整 Edge UAV 参数

**目标**：改变 UAV 飞行能耗权重

**位置**：`config/config.py`

```python
# ENERGY_PARAMS section
UAV_ENERGY_WEIGHT = 0.3  # 增大权重 = 优先降低能耗
```

### 任务 4：分析运行结果

**目标**：检查最新运行是否满足 S1-S4 标准

**步骤**：
```bash
# 找最新运行
ls -la discussion/ | tail -5

# 分析结果
uv run python scripts/analyze_results.py --run-dir discussion/LATEST_RUN/
```

**S1-S4 标准**：
- **S1**: 程序正常退出 + JSON 文件存在
- **S2**: ≥1/3 个体成功使用自定义目标（`llm_status="ok"`）
- **S3**: ≥1 个个体的解可行（`feasible=true`）
- **S4**: 对照组基线分数已记录

### 任务 5：运行 Phase⑥ Step3 SOCP 修复

**目标**：执行轨迹优化的 DCP 非凸性修复

**前置**：
- 备份当前代码：`git commit -m "checkpoint: before Step3"`
- 读完整计划：`plans/phase6-step3-socp-fix-plan.md`

**步骤**（按计划 7 步执行）：
```bash
# Step1: 修改约束表达式
# ... 编辑 edge_uav/model/trajectory_opt.py

# Step3: 单元测试修改
pytest tests/test_trajectory_opt.py::test_socp_comm_constraint -v

# 全体测试
pytest tests/test_trajectory_opt.py -v
```

## 仿真参数详解

### 时间相关

| 参数 | 值 | 说明 |
|------|------|------|
| `TIME_SLOT_DURATION` | 1.0 (s) | 单个时间槽长度 |
| `PLANNING_HORIZON` | 10 | 规划的时间槽数 |
| **总规划时长** | 10s | = 10 × 1.0 |

### 实体数量

| 参数 | 值 | 说明 |
|------|------|------|
| `NUM_USERS` | 10-50 | 用户数（任务源） |
| `NUM_UAVS` | 2-5 | UAV 数量 |
| `NUM_EDGE_SERVERS` | 1-2 | 边缘服务器 |

### 能耗参数

在 `config/config.py` 的 `ENERGY_PARAMS`：

```python
DEVICE_POWER = 1.0      # W - 移动设备功率
SERVER_POWER = 5.0      # W - 服务器功率
UAV_POWER = 2.0         # W - UAV 功率
```

### 通信参数

在 `config/config.py` 的 `COMM_PARAMS`：

```python
PATH_LOSS_EXPONENT = 2.5    # 信道衰落指数
BANDWIDTH = 10e6            # Hz - 频带宽度
NOISE_POWER = -90           # dBm - 噪声功率
```

## 数据输出结构

### 运行结果位置

```
discussion/
└─ {run_id}/                      # 时间戳，如 20260325_152149
    ├─ population_result_0.json   # 初始种群
    ├─ population_result_1.json   # 第 1 代
    ├─ population_result_2.json   # 第 2 代
    └─ ... 更多代数
```

### JSON 结构（每个个体）

```json
{
  "individual_id": 0,
  "prompt": "... LLM 提示词 ...",
  "generated_objective": "... LLM 生成的代码 ...",
  "llm_status": "ok",                    // "ok" 或错误信息
  "used_default_obj": false,             // 是否使用默认目标
  "feasible": true,                      // 解是否可行
  "evaluation_score": 1234.56,           // 适应度分数
  "generation": 0                        // 第几代
}
```

### 分析命令

```bash
# 查看某代的结果摘要
uv run python scripts/analyze_results.py --run-dir discussion/20260325_152149/

# 输出示例：
# Run: 20260325_152149 (Generation: 10)
# S1: PASS (exit 0 + JSON exist)
# S2: PASS (7/10 individuals with llm_status=ok)
# S3: PASS (10/10 individuals feasible)
# S4: PASS (baseline recorded)
```

## 性能指标

| 指标 | 范围 | 说明 |

## 常见问题排查

### "LLM API 连接失败"

```bash
# 检查连接
uv run python scripts/check_llm_api.py

# 如果失败，检查：
# 1. config/env/.env 是否存在且凭证正确
# 2. API 代理是否在线
# 3. 网络连接是否正常
```

### "优化超时"

```bash
# 增加超时时间（testEdgeUav.py 第 600s 超时）
# 或降低问题规模：
HS_POP_SIZE=3 HS_ITERATION=5 uv run python scripts/testEdgeUav.py
```

### "DCP 约束违反"（Phase⑥ Step3）

```bash
# 这是已知问题，按 Step3 计划修复
# 详见 plans/phase6-step3-socp-fix-plan.md
```

### "测试失败"

```bash
# 运行具体的测试看详情
pytest tests/test_trajectory_opt.py::test_name -v -s

# 查看哪些测试通过/失败
pytest tests/ --tb=short | grep -E "PASSED|FAILED"
```
