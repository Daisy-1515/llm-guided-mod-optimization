# 当前提示词清单（2026-04-03）

## 1. 结论摘要

当前仓库中有两套提示词体系：

1. **Edge-UAV（当前主链路，默认在用）**
2. **Taxi/MoD（历史链路，保留兼容）**

默认开发与实验入口是 Edge-UAV 管道（`scripts/run_edge_uav.py`），不是旧 Taxi 管道。

---

## 2. Edge-UAV 提示词（主链路）

### 2.1 文件位置

- `edge_uav/prompt/base_prompt.py`
- `edge_uav/prompt/mod_prompt.py`

### 2.2 基础组件（`EdgeUavPrompts`）

`base_prompt.py` 定义了 5 个核心提示词组件：

1. `prompt_scenario`：系统与任务背景
2. `prompt_static_info`：场景统计信息（运行时填充）
3. `prompt_init_level1model`：`OffloadingModel` 源码与结构说明
4. `prompt_obj_format`：LLM 输出 JSON 规范（`obj_description` + `obj_code`）
5. `prompt_cons_restriction`：Gurobi 编码约束与变量使用规则

说明：

- `set_scenario_info(tasks, uavs, time_slots)` 会写入运行期统计摘要
- 约束规则中已包含 math 非线性系数变换鼓励（如 `exp/log1p/sqrt/tanh/pow`，作用于常量系数）

### 2.3 进化策略（`EdgeUavModPrompts`）

`mod_prompt.py` 定义了 4 种生成方式：

1. `get_prompt_way1`：全新生成（random-explore）
2. `get_prompt_way2`：基于最佳个体局部改进（local-refine）
3. `get_prompt_way3`：结构性重构（structural-mutate）
4. `get_prompt_way4`：资源感知专项生成（resource-aware-specialization）

### 2.4 实际路由（运行时）

在 `heuristics/hsIndividualEdgeUav.py` 的 `getNewPrompt()` 中：

- `WAY_RANDOM` -> `get_prompt_way1`
- `WAY_MEMORY` -> `get_prompt_way2`
- `WAY_PITCH` -> `get_prompt_way3`
- `WAY_CROSS` -> `get_prompt_way4`
- `default` 或异常 -> 回退默认目标函数（不走 LLM）

---

## 3. Taxi/MoD 提示词（历史链路）

### 3.1 文件位置

- `prompt/basicPrompt.py`
- `prompt/modPrompt.py`

### 3.2 基础组件（`basicPrompts`）

核心结构与 Edge-UAV 类似，但面向 Taxi/Passenger 旧模型：

- `prompt_scenario`
- `prompt_static_map_info`
- `prompt_init_level1model`
- `prompt_obj_format`
- `prompt_cons_restriction`
- 可选 `prompt_future_impacts`

额外开关：

- `llmHasMemory`
- `llmHasFuture`

### 3.3 进化策略（`modPrompts`）

`modPrompt.py` 提供 3 种方式：

1. `get_prompt_way1`
2. `get_prompt_way2`
3. `get_prompt_way3`

在 `heuristics/hsIndividual.py` 中被调用（旧链路）。

---

## 4. 配置与入口关系

### 4.1 当前默认入口

- `scripts/run_edge_uav.py`（主入口）
- `scripts/run_all_experiments.py`（批量实验入口）

### 4.2 提示词相关配置

- `config/setting.cfg` 的 `[promptSettings]`：
  - `mapPath`
  - `modelPath`

该配置主要对应旧 Taxi 提示词链路。

Edge-UAV 链路的模型路径在代码里固定拼装：

- `heuristics/hsIndividualEdgeUav.py` -> `edge_uav/model/offloading.py`

---

## 5. 快速定位命令（只读）

```powershell
rg -n "get_prompt_way|prompt_|objective_instruction" edge_uav/prompt prompt
rg -n "from edge_uav.prompt.mod_prompt|from prompt.modPrompt" heuristics
rg -n "run_edge_uav|run_all_experiments" README.md scripts
```

---

## 6. 维护建议

1. 新增提示词策略时，优先在 `edge_uav/prompt/mod_prompt.py` 扩展并保持 way 路由可追踪。
2. 若调整输出 JSON 结构，需同步检查 `hsIndividualEdgeUav` 的解析与回退逻辑。
3. 旧 `prompt/` 建议标注“legacy only”，避免与主链路混用。
