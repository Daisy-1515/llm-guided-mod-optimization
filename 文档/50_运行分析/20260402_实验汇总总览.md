## 0. 记录规范与状态定义

### 对比组定义

| 组别 | 含义 |
|---|---|
| A | LLM + HS |
| B | LLM-only |
| C1 | random + HS |
| C2 | parametric + HS |
| D1 | fixed objective |
| D2 | fixed + HS（手工参数） |

### 状态定义

- **完整**: `manifest.json` 存在，`run_seed_*.json` 数量达到 `groups × seeds`，且有 `comparison_summary.json`。
- **部分**: 有可读运行结果，但缺 `comparison_summary.json` 或 run 文件数量不足。
- **异常**: 缺关键文件或关键文件不可解析（本次无）。

### 关键字段定义

- **seeds**: 运行种子列表（来自 manifest / run_seed 文件）。
- **numTasks / numUAVs / T**: 场景规模参数。
- **best_mean / best_min / feasible_mean / time_mean_s**: 以目录内全部 `run_seed_*.json` 聚合得到。
