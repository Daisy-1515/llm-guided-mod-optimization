## 一、章节推进模板

### 节首引入
```
本节设计一种[目标]优化算法。
首先，采用[方法A]将[长期问题]解耦为[单时隙问题]；
然后将[单时隙问题]解耦为[N]个优化子问题并分别求解。
具体而言，采用[方法B]求解[子问题1]；
然后通过[方法C]求解[子问题2]。
通过交替迭代优化，可获得原问题可行解。
```

### 问题简化引入
```
给定可行的[固定变量]，并移除无关项，[目标函数]可简化为：
[公式]
```

### 问题定性
```
[子问题名]仍为混合整数非线性、非凸优化问题。
[子问题名]是一个带有约束的凸优化问题。
```

### 方法引入
```
为求解该问题，本节采用[方法]，通过将[约束/结构]转化为[新形式]，将原问题转化为[新问题类型]。
```

### 子问题收敛
```
此循环持续进行，直至算法迭代次数到达最大迭代次数
或者目标函数的变化量小于预设阈值，则视为算法收敛并得到[变量]的可行解。
```

## 六、算法伪代码格式

```
算法X.X：[算法名称]
输入：[变量列表]
输出：[变量列表]
1：初始化：[变量列表]
2：while [条件] do
3：    for [循环变量] do
4：        根据公式（X-X）求得[变量]；
5：    end for
6：    [操作描述]；
7：    [变量]；
8：end while
9：return [输出变量]
```

**规范**：
- 步骤编号左对齐，用冒号分隔
- 中文注释直接内嵌，不另起行
- `根据公式（X-X）求得` 为标准引用句式
- 收敛判断写在 `while` 条件中，不单独作为 `if` 块

## 八、子问题分析写作流程（推荐顺序）

1. **固定变量 + 简化问题**：`给定可行的...，并移除无关项，可简化为：`
2. **定性问题**：凸/非凸、连续/整数、可行域性质
3. **引入求解方法**：罚函数 / KKT / 二分法 / SCA / LLM
4. **推导最优条件**：对目标求偏导 → 闭式解或迭代格式
5. **分类讨论**：约束有效 / 无效两种情况
6. **算法伪代码**：输入→初始化→迭代→输出
7. **复杂度分析**：各模块分析 → 总复杂度

---

## 九、风格边界（不做的事）

- 不使用"大幅""显著优于""革命性"等夸张表达
- 不在子问题推导中跳过"给定...，并移除无关项"的过渡
- 不省略分类讨论（约束有效时的解 vs 无效时的二分法）
- 不单独炫技公式，每个公式前必须有一句自然语言引导
- 复杂度分析必须拆解到每个子算法，再合并
