## 1. 定位

场景生成器是 Edge UAV 项目依赖链中的 **第 ② 步**，位于"① 数据结构 + ①.5 Config"之后、"③ 预计算模块"之前。

它的职责只有一件事：**出题**——生成一组终端设备、无人机和时间片，交给后面的模块去优化。本身不做任何优化计算。

```
① 数据结构 + Config（✅）
        ↓
② 场景生成器（本方案）  ← 你在这里
        ↓
③ 预计算模块（D_hat_local / D_hat_offload / E_hat_comp）
        ↓
④ hsIndividual 适配 + stub Level2
```

## 2. 场景生成器要生成什么

### 2.1 终端设备（ComputeTask）

在 `[0, x_max] × [0, y_max]` 的地图上随机撒 `numTasks` 个地面终端设备。每个设备带以下参数：

| 字段 | 含义 | 生成方式 |
|---|---|---|
| `index` | 设备编号 | 从 0 递增 |
| `pos` | 位置 `(x, y)` | 地图范围内均匀随机，**2D**（高度不在这里） |
| `D_l` | 上行数据量 (bits) | 从 `[D_l_min, D_l_max]` 均匀采样 |
| `D_r` | 下行数据量 (bits) | 从 `[D_r_min, D_r_max]` 均匀采样 |
| `F` | 计算量 (cycles) | 从 `[F_min, F_max]` 均匀采样 |
| `tau` | 截止时间 (s) | 从 `[tau_min, tau_max]` 均匀采样 |
| `f_local` | 本地 CPU 频率 (Hz) | 从 config 读取默认值 |
| `active[t]` | 时隙活跃标志 | 连续窗口模式，见 2.3 |

### 2.2 无人机（UAV）— 固定基站方案

所有无人机从**同一个基站**出发，任务结束后**必须飞回基站**。

| 字段 | 含义 | 生成方式 |
|---|---|---|
| `index` | 无人机编号 | 从 0 递增 |
| `pos` | 起飞位置 | **= depot_pos**（基站坐标，从 config 读取） |
| `pos_final` | 降落位置 | **= depot_pos**（回基站） |
| `E_max` | 电池能量 (J) | 从 config 读取 |
| `f_max` | 最大 CPU 频率 (Hz) | 从 config 读取 |
| `N_max` | 每时隙最大承载任务数 | `None`（不限制） |

**固定基站的含义**：
- 现实中无人机从机巢/充电站起飞，服务完回机巢
- `depot_pos` 默认 `(x_max/2, y_max/2)`（地图中心），可在 config 中配置
- 3 架无人机都从同一个点出发、回同一个点
- Level 2 轨迹优化的约束：`q_j(t=0) = depot_pos`，`q_j(t=T) = depot_pos`

### 2.3 时隙活跃标志（active[t]）

不是所有设备在 20 秒全程都有任务。每个设备只在一段**连续时间窗口**内活跃：

```
设备 0: ____████████____    （第4~11秒活跃）
设备 1: ██████__________    （第0~5秒活跃）
设备 2: ________████████    （第8~15秒活跃）
...
```

生成规则：
- 窗口长度从 `[window_min, window_max]` 均匀采样
- 窗口起点随机，但保证 `start + length <= T`
- **保证每个设备至少 1 个时隙活跃**
- 用 `defaultdict(bool)` 实现，未填充的时隙自动返回 `False`

### 2.4 时间片列表

```python
time_slots = [0, 1, 2, ..., T-1]  # T=20 → 20个时间片
```

时间索引约定（**全项目统一**）：
- 从 0 开始，步长 1
- `time_slots = list(range(T))`
- `pos_final` 不对应某个时隙，是无人机的回收位置约束

## 3. 返回值结构

定义 `EdgeUavScenario` dataclass 作为返回值：

```python
@dataclass
class EdgeUavScenario:
    tasks: dict[int, ComputeTask]    # {0: task0, 1: task1, ...}
    uavs: dict[int, UAV]            # {0: uav0, 1: uav1, ...}
    time_slots: list[int]           # [0, 1, ..., T-1]
    seed: int                       # 随机种子（可复现）
    meta: dict                      # 元信息：T, delta, x_max, y_max, H, depot_pos 等
```

**不包含** `D_hat_local`、`D_hat_offload`、`E_hat_comp` —— 这些属于 ③ 预计算模块。

`edgeUavScenarioGenerator.py`（项目根目录，与原 `scenarioGenerator.py` 并行共存）

### 5.2 类结构

class EdgeUavScenarioGenerator:
    """Edge UAV 场景生成器。"""

    def __init__(self):
        pass

    def getScenarioInfo(self, config) -> EdgeUavScenario:
        """主入口：读 config，生成场景，返回 EdgeUavScenario。"""
        ...

    # --- 私有方法 ---
    def _generate_time_slots(self, T: int) -> list[int]:
        ...

    def _generate_tasks(self, num_tasks, config, rng) -> dict[int, ComputeTask]:
        ...

    def _generate_uavs(self, num_uavs, config, rng) -> dict[int, UAV]:
        ...

        """场景可行性校验。"""

### 5.3 不继承原项目 TaskGenerator

理由：原 TaskGenerator 的逻辑（CSV 加载、路网距离矩阵、乘客/出租车）与 Edge UAV 完全无关。

### 5.4 不替换原 `scenarioGenerator.py`

原文件保留不动，两套场景生成器并行存在。旧的 HS 链路（testAll.py → hsFrame.py）不受影响。

## 6. 场景可行性校验

生成完场景后，必须做以下检查：

| 检查项 | 规则 | 不通过的处理 |
|---|---|---|
| 活跃时隙 | 每个设备至少 1 个 `active[t] = True` | 断言失败 |
| 索引唯一 | 任务和 UAV 的 index 不重复 | 断言失败 |
| 位置边界 | 所有 pos 在 `[0, x_max] × [0, y_max]` 内 | 断言失败 |
| tau 可行性 | 粗估卸载时延，若 >80% 任务无法卸载则告警 | 告警 + 可选重采样 |

**tau 可行性** 的具体逻辑：用最近 UAV 的距离粗算一次 `D_hat_offload`，如果大部分任务的卸载时延都超过 `tau`，说明题出得太难、无人机形同摆设。此时打印警告，让用户决定是否放宽 `tau_min`。

## 7. 序列化（save/load）

```python
# 保存
scenario.save_json("instances/edge_uav/scenario_seed42.json")

# 加载
scenario = EdgeUavScenario.load_json("instances/edge_uav/scenario_seed42.json")
```

目的：固定场景、横向对比不同算法。相同 seed 生成的场景完全一致。

## 9. 不在本阶段做的（v1.1 增强）

- 热点区域 / 非均匀任务分布
- 异构 UAV（不同 E_max / f_max）
- `active_mode = bernoulli`（随机离散活跃）
- 自由终点模式（`pos_final ≠ depot_pos`）
- 替换旧 `scenarioGenerator.py` 或改旧 HS 链路

---

## 10. 风险与注意事项

| 风险 | 说明 | 缓解措施 |
|---|---|---|
| tau 过激 | 截止时间太短 → 全部退化为本地计算 | 可行性校验 + 告警 |
| off-by-one | 时间索引约定不统一 | 本文档冻结：`range(T)`，从 0 开始 |
| 接口不兼容 | 新场景格式与旧 HS 链路冲突 | 不动旧文件，并行共存 |
| 2D/3D 混淆 | 位置只放 (x,y)，高度 H 在全局 config | 生成器不碰 3D |
| 全局随机污染 | 生成器改了全局 seed 影响其他模块 | 用独立 `numpy.random.Generator` |
