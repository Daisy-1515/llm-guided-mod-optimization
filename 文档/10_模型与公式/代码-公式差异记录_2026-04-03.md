## 差异 1：L1-C2 — 硬约束 vs 软惩罚

### 公式文档（2026-03-24）

L1-C2 定义为硬约束：

```
x_{i,j}^t = 0,  if D̂_{i,j}^t > τ_i
```

语义：禁止将任务卸载到预估执行时延超过截止期 τ_i 的 UAV；
若所有卸载目标均超时，则触发 drop 失败机制（见差异 2）。

### 当前代码（offloading.py）

`setupVars`（第 214-238 行）：为所有活跃任务**无条件**创建 `x_local` 和 `x_offload` 变量，
不按 D̂ 过滤卸载目标：

```python
# offloading.py:214-238 — setupVars
for i in self.taskList:
    for t in self.timeList:
        if not self.task[i].active[t]:
            continue
        self.x_local[i, t] = self.model.addVar(...)   # 无条件创建
        for j in self.uavList:
            self.x_offload[i, j, t] = self.model.addVar(...)  # 无条件创建
```

`setupCons`（第 242-279 行）：只含 L1-C1（分配唯一性）和 L1-C3（承载量上限），
**不含 L1-C2**：

```python
# offloading.py:242-263 — setupCons（只建 L1-C1 约束）
for i in self.taskList:
    for t in self.timeList:
        if not self.task[i].active[t]:
            continue
        offload_sum = gb.quicksum(...)
        self.model.addConstr(self.x_local[i, t] + offload_sum == 1, ...)
```

### 差异说明

| 维度 | 公式文档（2026-03-24）| 当前代码（2026-04-01）|
|------|----------------------|----------------------|
| L1-C2 存在 | 是（硬约束） | **否**（已删除）|
| 超时处理 | 禁止卸载；全不可行时 drop | 超时通过目标值 D/τ > 1 自然惩罚（软惩罚）|
| `_offload_feasible` 方法 | 用于建立 L1-C2 约束 | 方法仍在（第 200-205 行），但 **setupVars 中未调用** |

**设计意图**（见 `objectives.py:30-33` 注释）：

> 超时通过 D/τ > 1 自然反映在目标值中。

## 差异 2：drop 失败机制

### 公式文档（2026-03-24）

文档第 452-454 行描述 drop 变量和 penalty_drop 惩罚项：

- `x_drop_{i,t}` ∈ {0,1}：任务 (i,t) 被丢弃标记
- 目标函数：`cost1 + cost2 + cost3`，其中 `cost3 = penalty_drop · Σ x_drop`
- `getOutputs` 返回格式：`{t: {"local": [], "offload": {}, "drop": []}}`

### 当前代码（commit 2c53c0e，2026-04-01）

drop 机制**已完全移除**：

**`getOutputs`**（`offloading.py:154-189`）：

```python
result[t] = {"local": local_tasks, "offload": offload_tasks}
# 无 "drop" 字段
```

**`CANONICAL_OBJECTIVE_V1`**（`objectives.py:14-34`）：

```
minimize: Σ_{i∈I} Σ_{t∈T} [ α·D̂_{i,⊙}(x) / τ_i
                             + γ_w·Ê_{comp}(x,f_edge) / E_{max,edge} ]
```

仅 cost1（时延）+ cost2（能耗），无 cost3（drop 惩罚）。

### 差异说明

| 维度 | 公式文档（2026-03-24）| 当前代码（2026-04-01）|
|------|----------------------|----------------------|
| drop 变量 | 存在 | **已删除** |
| 目标函数项 | cost1 + cost2 + cost3 | cost1 + cost2 |
| getOutputs 格式 | 含 `"drop"` 字段 | **无 `"drop"` 字段** |
| 失败语义 | 任务可被显式丢弃 | 任务不被丢弃，高时延自然惩罚 |
