# 聚合归一化实现方案

> 基于《公式与两层解耦整合版_最新版_2026-03-24.md》的系统成本公式 (20) 完整实现
> 生成日期：2026-04-05

---

## Context

### 目的

根据文档《公式与两层解耦整合版_最新版_2026-03-24.md》，实现系统成本公式 (20) 的完整聚合归一化。

### 问题

当前代码缺少 `N_act` 和 `N_fly` 聚合归一化因子，导致：
- Level 1/2a/2b 目标函数尺度与文档不一致
- 三类成本块（时延、边缘能耗、飞行能耗）在求和后不可比
- Level 2b 可能因权重比例偏差影响轨迹优化结果

### 理论影响

| 层级 | 影响 |
|------|------|
| Level 1 | 归一化因子不改变最优决策（线性缩放） |
| Level 2a | 频率 KKT 解不受影响（线性缩放） |
| Level 2b | `N_act` vs `N_fly` 改变通信延迟与飞行能耗的相对权重，**可能影响轨迹** |

---

## 文档公式要求

根据公式 (20)，系统成本定义为：

$$N_{\mathrm{act}}=\sum_{t\in\mathcal{T}}\sum_{i\in\mathcal{I}}\zeta_i^t$$

$$N_{\mathrm{fly}}=|\mathcal{U}|(T-1)$$

目标函数各层需包含聚合归一化：

| 层级 | 文档公式 |
|------|----------|
| Level 1 | `(1/N_act) × [α·D̂/τ + γ·Ê_comp/E_max]` |
| Level 2a | `(1/N_act) × [α·F/(f·τ) + γ·γ_j·f²·F/E_max]` |
| Level 2b | `(α/N_act)·Σ(D/τ) + (λ/N_fly)·Σ(E_fly/E_max)` |

## 核心改动方案

### 1. 新增 N_act 和 N_fly 定义

**文件**：`edge_uav/model/precompute.py`

**公式**：
```
N_act = Σ_t Σ_i ζ_i^t  (全时域 active task-slot 总数)
N_fly = |U| × (T-1)    (全时域 UAV 移动段总数)
```

**改动点**：

1. `PrecomputeResult` 数据类新增字段：
```python
@dataclass(frozen=True)
class PrecomputeResult:
    D_hat_local: Scalar2D
    D_hat_offload: Scalar3D
    E_hat_comp: Scalar3D
    diagnostics: dict[str, Any]
    # 新增
    N_act: int          # 全时域 active task-slot 总数
    N_fly: int          # 全时域 UAV 移动段总数
```

2. `precompute_offloading_inputs()` 计算并返回：
```python
N_act = sum(
    1 for i in scenario.tasks
    for t in scenario.time_slots
    if scenario.tasks[i].active[t]
)
N_fly = len(scenario.uavs) * (len(scenario.time_slots) - 1)
```

**复用现有逻辑**：`diagnostics["active_task_slots"]` 已跟踪活跃任务，可直接复用。

def default_dynamic_obj_func(self):
    inv_N_act = 1.0 / self.N_act

    cost1 = gb.quicksum(
        inv_N_act * self.alpha * self.D_hat_local[i][t] / self.task[i].tau
        * self.x_local[i, t]
        ...
    ) + gb.quicksum(
        inv_N_act * self.alpha * self.D_hat_offload[i][j][t] / self.task[i].tau
        * self.x_offload[i, j, t]
        ...
    )

    cost2 = gb.quicksum(
        inv_N_act * self.gamma_w * self.E_hat_comp[j][i][t] / self.uav[j].E_max
        * self.x_offload[i, j, t]
        ...
    )
```

    ...
```

2. `_compute_objective()` 添加因子：
```python
def _compute_objective(
    f_edge, offload_sets, scenario, params,
    *, alpha, gamma_w, inv_N_act=1.0
) -> float:
    obj = 0.0
    for j, slot_map in offload_sets.items():
        for t, task_ids in slot_map.items():
            for i in task_ids:
                f = task_freqs.get(i, {}).get(t)
                obj += inv_N_act * alpha * task.F / (f * task.tau)
                obj += inv_N_act * gamma_w * params.gamma_j * f * f * task.F / uav.E_max
    return obj
```

**注意**：频率 KKT 解不受 inv_N_act 影响（线性缩放因子）。

2. `_build_sca_subproblem()` (Lines 683-887)：

**通信延迟 surrogate 项**：
```python
obj_comm_surrogate = cp.Constant(0)
for j, i, t in active_offloads:
    ...
    # 添加 inv_N_act 和 1/tau_i
    obj_comm_surrogate = obj_comm_surrogate \
        - inv_N_act * (D_l_i / r_up_ref ** 2) * r_up_lb / tau_i \
        - inv_N_act * (D_r_i / r_down_ref ** 2) * r_down_lb / tau_i
```

**推进能耗项** — 需要 E_max 归一化：
```python
for j in uavs:
    for t in range(T - 1):
        power_ub = term1 + term2_ub + term3_ub
        energy_contrib = power_ub * delta
        # 添加 inv_N_fly 和 E_max 归一化
        obj_propulsion += inv_N_fly * energy_contrib / uavs[j].E_max
```

3. `_evaluate_true_objective()` (Lines 979-1044)：

```python
def _evaluate_true_objective(
    scenario, q, traj_params, params, active_offloads,
    *, alpha=1.0, lambda_w=1.0, inv_N_act=1.0, inv_N_fly=1.0
) -> tuple[float, float, float]:
    ...
    # 通信延迟
    total_comm = 0.0
    for j, i, t in active_offloads:
        ...
        total_comm += inv_N_act * (D_l_i / r_up + D_r_i / r_down) / tasks[i].tau

    # 推进能耗 — 添加 E_max 归一化
    E_prop_normalized = sum(
        inv_N_fly * per_uav_energy[j] / scenario.uavs[j].E_max
        for j in per_uav_energy
    )

    total_obj = alpha * total_comm + lambda_w * E_prop_normalized
    return float(total_obj), float(total_comm), float(E_prop)
```

## 验证方案

### 单元测试

1. **N_act/N_fly 计算正确性**
   - 构造已知 active task 数量的场景
   - 验证 `N_act = 预期值`
   - 验证 `N_fly = |U| × (T-1)`

2. **归一化因子传递**
   - 验证 `PrecomputeResult.N_act` 正确传递到各层
   - 验证目标函数中归一化因子正确应用

3. **目标值尺度验证**
   - 对比启用/禁用归一化前后的目标值
   - 验证目标值比例为 `1/N_act` 或 `1/N_fly`

### 集成测试

1. **BCD 收敛性**
   - 验证归一化后 BCD 仍能收敛
   - 对比归一化前后迭代次数

2. **最优解不变性验证**
   - Level 1 和 Level 2a：验证决策变量不变
   - Level 2b：注意权重比例变化可能影响轨迹

### 文档一致性验证

- 验证实现与文档公式 (20) 完全一致
- 验证各级目标函数符号和系数正确
