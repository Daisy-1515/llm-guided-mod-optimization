### 3.2 空地信道与双向通信模型（2-3 页）

**内容要点**：

1. **空地信道模型**
   - 三维距离：d_{i,j}^t = sqrt(H² + ||p_i - q_j^t||²)
   - 信道增益：g_{i,j}^t = ρ₀ / (d_{i,j}^t)²
   - 说明 LoS 假设的适用条件（UAV 高度 ≥ 100m）

2. **上行通信模型**
   - 上行速率：r_{i,j}^{up,t} = B_up · log₂(1 + P_i · g_{i,j}^t / N₀)
   - 上行传输延迟：T_{i,j}^{up,t} = D_l^i / r_{i,j}^{up,t}
   - 物理含义：终端将原始计算数据上传至 UAV

3. **下行通信模型**
   - 下行速率：r_{j,i}^{down,t} = B_down · log₂(1 + P_j · g_{j,i}^t / N₀)
   - 下行传输延迟：T_{j,i}^{down,t} = D_r^i / r_{j,i}^{down,t}
   - 物理含义：UAV 将计算结果回传至终端

4. **上下行不对称性分析** ★
   - 数据量不对称：D_l >> D_r（原始数据 vs 计算结果）
   - 功率不对称：P_i ≠ P_j（终端功率受限 vs UAV 功率充足）
   - 时空不对称：上行/下行阶段 UAV 位置不同，信道增益不同

**符号规范**：数据量用 D，延迟用 T，速率用 r，增益用 g

### 3.3 任务卸载与边缘计算模型（3-4 页）

**内容要点**：

1. **计算任务特征化**
   - 任务五元组：(D_l, D_r, F, τ, active)
   - D_l：上行输入数据量（bits）
   - D_r：下行结果数据量（bits）
   - F：所需 CPU 周期数（cycles）
   - τ：最大容忍时延（deadline）
   - active(t)：时隙活跃标识

2. **本地计算模型**
   - 本地执行延迟：T_{i}^{local} = F_i / f_i^{local}
   - f_i^{local}：终端本地 CPU 频率
   - 适用场景：任务较小或无可用 UAV 时

3. **边缘计算模型**
   - 边缘执行延迟：T_{i,j}^{comp} = F_i / f_{j,i}
   - f_{j,i}：UAV j 分配给任务 i 的 CPU 频率
   - CPU 频率约束：Σ_i f_{j,i} ≤ f_max（UAV 计算资源有限）

4. **双向卸载总延迟** ★ 核心公式
   - T_{i,j}^{total} = T_{i,j}^{up} + T_{i,j}^{comp} + T_{j,i}^{down}
   - 三段串行：上传 → 计算 → 回传
   - 因果约束：下行传输起始时间 ≥ 上行结束时间 + 计算时间

5. **卸载决策与时限约束**
   - 二值决策变量：x_{i,j}^t ∈ {0, 1}（是否卸载到 UAV j）
   - 唯一分配约束：每个活跃任务恰好分配一个执行目标
   - 时限约束：T_{i,j}^{total} ≤ τ_i

6. **关联与调度约束**
   - 用户-UAV 关联：基于通信可达性
   - UAV 容量约束：同一时隙服务任务数上限

### 3.4 无人机轨迹与系统能耗模型（3-4 页）

**内容要点**：

1. **2D 轨迹模型**
   - 离散化运动方程：q_j^{t+1} = q_j^t + v_j^t · δ
   - q_j^t = [x_j^t, y_j^t]^T：UAV j 在时隙 t 的水平位置
   - 速度为派生量：v_j^t = ||q_j^{t+1} - q_j^t|| / δ

2. **运动约束集**
   - C-T1：边界约束 0 ≤ x_j^t ≤ x_max, 0 ≤ y_j^t ≤ y_max
   - C-T2：首末位置约束 q_j^1 = q_j^I, q_j^T = q_j^F
   - C-T3：最大速度约束 ||q_j^{t+1} - q_j^t|| ≤ v_max · δ
   - C-T4：末端可达性约束
   - C-T5：UAV 间安全距离约束 ||q_j^t - q_{j'}^t|| ≥ d_safe

3. **旋翼推进能耗模型**
   - 推进功率公式（四项）：
     P_prop = η₁(1 + 3v²/v_tip²) + η₂√(√(η₃ + v⁴/4) - v²/2) + η₄v³
   - 飞行能耗：E_j^{fly,t} = P_j^{prop,t} · δ
   - 备注：第二项为非凸项（凹函数），详细推导见附录

4. **边缘计算能耗模型**
   - 计算能耗：E_{j,i}^{comp} = γ_j · (f_{j,i})² · F_i
   - 与 CPU 频率平方成正比（凸函数）
   - 频率-能耗权衡：高频率减少延迟但增加能耗

5. **系统能量预算约束**
   - 总能耗 = 飞行能耗 + 计算能耗
   - Σ_t E_j^{fly,t} + Σ_i Σ_t E_{j,i}^{comp,t} ≤ E_max

### 3.5 联合优化问题建模（2-3 页）

**内容要点**：

1. **决策变量汇总**
   - 离散变量：x_{i,j}^t ∈ {0, 1}（卸载决策）
   - 连续变量：f_{j,i}^t（CPU 频率分配）、q_j^t（UAV 轨迹）
   - 变量维度分析

2. **多目标加权目标函数**
   - min C = α · C_delay + γ_w · C_comp + λ_w · C_fly
   - C_delay = Σ归一化延迟（延迟/deadline）
   - C_comp = Σ归一化计算能耗（能耗/E_max）
   - C_fly = Σ归一化飞行能耗（能耗/E_max）
   - 引用前文公式编号，不重复推导

3. **约束条件汇总**（引用前文编号）
   - 通信约束：引用 3.2 节公式
   - 计算约束：引用 3.3 节公式（唯一分配、时限、容量）
   - 运动约束：引用 3.4 节公式（C-T1 至 C-T5）
   - 能量约束：引用 3.4 节公式
   - 因果约束：双向数据流时序依赖

4. **问题分类与求解难点**
   - 问题类型：混合整数非线性规划（MINLP）
