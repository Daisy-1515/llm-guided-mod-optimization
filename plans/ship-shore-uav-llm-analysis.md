## 1. 背景与问题定义

### 1.1 研究动机

海上通信正面临 6G 时代的范式转型。传统船岸通信依赖卫星链路或岸基基站，受限于高延迟、低带宽和覆盖盲区。多无人机（Multi-UAV）作为空中中继/边缘计算节点，正成为扩展海上通信覆盖的关键技术。与此同时，大语言模型（LLM）在无线通信资源分配和网络管理中展现出优于传统深度学习 40%-80% 的性能提升。

**核心问题**：这两个快速发展的领域是否存在有价值的交叉点？LLM 能否成为海上多无人机通信系统的"智能决策引擎"？

## 2. 两个领域的独立综述

### 2.1 船岸背景下的多无人机通信

#### 核心挑战

- **远距离覆盖**：船岸距离可达数十至数百公里，远超单跳通信范围
- **动态拓扑**：船舶移动 + 海浪起伏 + UAV 机动 → 网络拓扑持续变化
- **带宽受限**：海上缺乏蜂窝基础设施，可用频谱资源有限
- **信道恶劣**：海面多径反射、蒸发波导效应、气象干扰

#### 主流技术方案（2025年）

1. **多UAV中继架构**：多架UAV组成空中中继链，将船岸通信范围从 8 英里扩展至 30 英里（DPI UMAR 系统）
2. **双UAV协作数据收集**：联合优化接入关系、UAV轨迹、数据卸载和通信资源分配，最小化系统能耗 ([Nature, 2025](https://www.nature.com/articles/s41598-025-10766-9))
3. **蒸发波导辅助通信**：利用海面蒸发波导实现 UAV 间远距中继 ([Nature, 2025](https://www.nature.com/articles/s41598-025-90065-5))
4. **内容缓存与网络编码**：UAV 中继系统集成缓存和网络编码，增强数据承载能力 ([IEEE, 2025](https://ieeexplore.ieee.org/document/11007515/))
5. **联合感知通信供能（JSCPB）**：UAV 同时执行感知、通信、无线充能和回传 ([arXiv, 2025](https://arxiv.org/abs/2505.12190))

#### 尚未解决的问题

- 大规模 UAV 集群的分布式协调
- 信道状态信息（CSI）的实时获取与预测
- 通信中断后的自治决策与恢复
- 非结构化海事数据（视频/雷达/AIS）的高效传输

### 2.2 大模型加速的通信机制

#### LLM 在无线通信中的范式转变

传统无线通信优化依赖数学规划或深度强化学习（DRL），面临以下瓶颈：
- 约束 NP-hard 问题求解困难
- 环境变化后需要大量重训练
- 缺乏自然语言级别的问题理解能力

LLM 的引入带来三个层面的革新：

| 层面 | 传统方法 | LLM 方法 | 代表工作 |
|------|---------|----------|---------|
| 问题建模 | 人工数学建模 | 自动检测非凸分量并转化 | [LLM-OptiRA](https://arxiv.org/abs/2505.02091) (96% 执行率) |
| 资源分配 | 固定 DL 架构 | Prompt-based 自适应优化 | [LLM-RAO](https://arxiv.org/abs/2502.02287) (提升 40%) |
| 网络管理 | 单任务独立优化 | 多智能体分解协作 | [Survey, 2025](https://arxiv.org/abs/2509.05946) |
| RL 增强 | 纯 DRL | LLM + RL 混合 | [LLM-RL, 2026](https://arxiv.org/html/2602.13210) (步骤减少 46%) |

#### LLM + UAV 的已有探索

- **UAVs Meet LLMs** ([arXiv, 2025](https://arxiv.org/html/2501.02341v1))：综述 LLM-UAV 集成的方法论、应用和挑战
- **分层协作 LLM 控制** ([arXiv, 2025](https://arxiv.org/abs/2506.06532))：HAPS 部署 LLM 管理 UAV-HAPS 连接 + UAV 动作控制
- **Agentic AI 多UAV轨迹优化** ([arXiv, 2025](https://arxiv.org/html/2508.16379))：基于 LLM+RAG 的 ARMAIT 框架
- **LLM 增强多智能体 RL 任务卸载** ([PMC, 2025](https://pmc.ncbi.nlm.nih.gov/articles/PMC11723390/))：LLM 用于区域分解 + GCN 自注意力

## 3. 统一问题建模与 LLM 角色定位

> 本节是连接两个独立领域的关键桥梁。

### 3.1 海上多UAV通信系统的统一建模框架

```
系统实体:
├── 岸基站 (Shore BS)     — 大算力，部署 LLM 推理服务
├── UAV 集群 (U 架)       — 空中中继 + 边缘计算节点
├── 船舶 (移动终端)        — 产生任务，位置动态变化
├── 海面浮标 (可选)        — 固定传感节点
└── 卫星/HAPS (可选)       — 备用回传链路

决策变量:
├── 离散: UAV-船舶关联 x_{i,j}^t, 中继路由选择
├── 连续: UAV 轨迹 q_j^t, 发射功率 P_j^t, CPU 频率 f_{j,i}^t
└── 带宽分配: B_{i,j}^t

目标函数: min { 加权延迟 + 能耗 + 通信中断率 }

时间尺度:
├── 毫秒级: 功率控制、时隙调度 → 传统优化/启发式
├── 秒~分钟级: 轨迹航向、中继选择 → LLM 决策层
└── 分钟~小时级: 宏观拓扑规划、任务编排 → LLM 策略层
```

### 3.2 LLM 在系统中的三种角色定位

| 角色 | 输入 | 输出 | 时间尺度 | 部署位置 |
|------|------|------|---------|---------|
| **策略编排器** | 任务描述 + 海况 + 船舶轨迹 | 宏观资源调配策略 | 分钟级 | 岸基 |
| **求解器编排器** | 优化问题描述 | 分解策略 + 初始解 + 约束转换 | 秒级 | 岸基 |
| **语义压缩器** | 原始传感数据 | 语义特征/意图描述 | 秒级 | UAV 端侧 (SLM) |

**关键架构约束**：
- UAV 机载算力不足以运行大模型 → 必须采用"岸基大模型 + UAV 端侧小模型(SLM/蒸馏模型)"的云边协同架构
- LLM 推理延迟（秒级）>> 信道相干时间（毫秒级）→ 必须采用两层时间尺度解耦
- LLM 存在幻觉风险 → 必须设置数学规划/安全屏障(Guardrail)作为约束验证层

### 3.3 与传统方法的增量价值论证

为什么必须引入 LLM，而非仅用 DRL/MARL/MPC/启发式？

| 挑战 | 传统方法的局限 | LLM 的增量价值 |
|------|--------------|---------------|
| 场景多样性 | DRL 每换场景需重训练 | Prompt-based 零/少样本适配 |
| 约束表达 | 人工建模，难以处理非结构化约束 | 自然语言约束理解与转化 |
| 异常处理 | 预定义规则，无法泛化 | 上下文推理 + 经验迁移 |
| 多目标权衡 | 固定权重或帕累托搜索 | 语义理解用户偏好动态调整 |
| 跨域知识 | 领域隔离 | 海洋气象 + 通信 + 飞控知识融合 |

## 4. 五个潜在创新研究方向

> 排序依据：创新性 × 可行性 × 海事场景区分度

### 方向 1 [主攻 ★★★] 海况驱动的时空拓扑预测与鲁棒轨迹联合优化

**评分**: 9.0/10

**核心思想**：利用 LLM 的序列推理能力，输入历史船舶 AIS 轨迹 + 气象预报 + 海况数据，预测未来通信拓扑演化，引导 UAV 集群进行鲁棒轨迹规划。

**创新点**：
- 传统轨迹优化假设终端固定或运动已知；海上场景船舶运动受海况影响，具有高度不确定性
- LLM 不直接做轨迹控制器，而是作为"时空拓扑预测器 + 策略编排器"
- 底层轨迹执行交给 MPC/凸优化求解器，确保运动约束满足

**技术路径**：
1. LLM 接收 { AIS 数据, 气象预报, 当前 UAV 状态 } → 输出"未来 N 分钟拓扑预测 + 建议航向"
2. 传统优化器以 LLM 输出为 warm start，求解约束轨迹优化
3. 安全屏障验证轨迹可行性（能量、碰撞、覆盖）

**与现有项目关联**：直接扩展当前 `edge_uav/model/offloading.py` 的轨迹优化部分，将静态终端替换为移动船舶

---

### 方向 2 [主攻 ★★★] LLM 引导的海上通信资源分配联合优化

**评分**: 8.8/10

**核心思想**：将 LLM-RAO / LLM-OptiRA 框架迁移到海上多 UAV 场景，LLM 作为"目标设计器 + 约束转换器 + 求解器编排器"。

**创新点**：
- 继承 LLM-RAO 的 prompt-based tuning 策略，适配海上通信特有约束（断连、海况、能量受限）
- LLM 自动识别海上通信问题中的非凸分量并转化为可解形式（扩展 LLM-OptiRA）
- 多目标权重根据海况语义描述动态调整（"暴风天气优先保证可靠性" vs "平静海面最大化吞吐量"）

**与现有项目关联**：最强关联。现有 `edge_uav/prompt/` 的 prompt 框架和 `llmAPI/` 接口可直接复用，仅需修改场景描述和约束条件

---

### 方向 3 [次主攻 ★★] LLM 驱动的语义感知多 UAV 中继路由与卸载

**评分**: 8.5/10

**核心思想**：传统中继路由基于 SNR 或队列长度。引入 LLM 后，根据船舶端产生任务的"语义优先级"进行差异化路由与卸载。

**创新点**：
- 语义对象明确化：AIS 文本消息 / 雷达点云摘要 / 搜救图像 / 气象数据 → 不同语义优先级
- 语义效用指标：不仅衡量比特率，还衡量"语义保真度"和"任务完成价值"
- 海上带宽极度受限 → 语义压缩的价值远大于陆地场景

**挑战**：需要定义可量化的语义效用函数，否则难以与传统速率型方案公平比较

---

### 方向 4 [扩展 ★] 基于 LLM 的多智能体断连恢复与容错机制

**评分**: 7.2/10

**核心思想**：海上 UAV-岸基链路断连是常态。LLM 不做闭环飞控，而是做"故障诊断 + 恢复策略生成"。

**创新点**：
- 断连时 UAV 端侧小模型基于岸基 LLM 的"最后意图"进行自治决策
- 多 UAV 在断连状态下共享局部算力池，维持基本服务
- LLM 重连后进行"差异同步 + 策略校正"

**风险**：审稿人会追问时延、可验证性和安全性。建议限定为策略建议层，执行交给可验证控制器

---

### 方向 5 [扩展 ★] 端云协同非结构化海事数据处理与卸载优化

**评分**: 7.0/10

**核心思想**：船舶产生大量非结构化数据（视频流/雷达点云/AIS 日志），UAV 上部署轻量 VLM 做初步语义提取，仅将高价值特征卸载回岸基。

**创新点**：
- 强绑定海事场景：搜救图像目标检测、海面目标跟踪、船载日志异常检测
- "算力换带宽"：用 UAV 端侧计算减少回传数据量
- 需要解释为什么需要 LLM/VLM 而非传统编码压缩

**风险**：范围过宽，容易变成泛化的边缘计算论文

- LLM 层：理解海况语义、编排宏观策略、生成代理目标函数
- 优化器层：满足严格的物理约束（能量、碰撞、覆盖、时延）
- 安全屏障：拦截 LLM 幻觉导致的不可行解

## 6. 风险与挑战

| 风险 | 影响 | 可能性 | 缓解策略 |
|------|------|--------|---------|

## 7. 核心结论

### 两者关系的本质

"船岸多无人机通信"与"LLM 加速通信机制"的关系是 **"场景拉动 + 技术赋能"** 的互补关系：

1. **场景拉动**：海上通信的极端动态性（移动船舶 + 恶劣信道 + 频繁断连）暴露了传统优化方法的三大不足——泛化差、适应慢、无法处理非结构化信息
2. **技术赋能**：LLM 恰好提供了 prompt-based 零样本适配、自然语言约束理解、跨域知识融合三项能力来弥补这些不足
3. **研究空隙**：两者的交叉几乎未被探索（仅 ICCCN 2025 一篇会议论文），构成显著的创新机会

### 推荐优先级

```
主攻（最高ROI）:
  ★★★ 方向 1: 海况驱动拓扑预测 + 鲁棒轨迹优化 (9.0)
  ★★★ 方向 2: LLM 引导海上通信资源分配 (8.8)

次主攻:
  ★★  方向 3: 语义感知中继路由 (8.5)

扩展方向:
  ★   方向 4: 断连恢复容错 (7.2)
  ★   方向 5: 非结构化数据处理 (7.0)
```
