### 轨迹类型判断

**结论：非直线轨迹（High Non-Linearity）**

#### 证据链
1. **极端速度跳跃 (1038 m/s ≈ 20.8x 限制)**
   - 相邻时隙间 UAV 位置变化剧烈
   - 若是直线轨迹，相邻时隙速度应连续、相近
   - 极端速度表明轨迹存在：
     - 大幅方向改变（转向角大）
     - 快速加速/减速
     - 可能的"弹跳"行为

2. **速度分布特点**
   ```
   速度范围: [50, 1038] m/s
   跨度: ~20x
   ```
   - 表明轨迹中有多个显著的"加速区间" → 多段轨迹
   - 非均匀速度 → 非直线

3. **违反位置**
   - UAV 0: 主要在 t=1,3,5,8,9... 处（分散）
   - UAV 1: 主要在 t=1,3,4,5,20,21... 处（分散）
   - UAV 2: 主要在 t=1,6,7,12... 处（分散）
   - **所有 UAV 都有多处违反** → 不是单一长线段，而是多段路径

### 可视化模型

若轨迹是直线：
```
t=0    t=1    t=2    t=3    t=4    t=5
 o-----o-----o-----o-----o-----o
```

## 3. BCD 循环可行性

### 当前状态
```
BCD 轨迹可行性: FAIL ✗
错误消息: check_trajectory_monotonicity failed (96 errors)
```

### 失败原因链
1. **Level-2b 轨迹优化器** 在优化通信延迟和飞行能耗时
2. **过度拟合** 到目标函数，忽视速度连续性约束
3. **结果轨迹** 包含不现实的速度跳跃（>1000 m/s）
4. **验证器** `check_trajectory_monotonicity` 拒绝此轨迹

### 改进方向
1. **增强约束**：在 L2 轨迹优化中加入速度平滑项
   ```
   minimize: ... + lambda * sum(|v[t+1] - v[t]|²)
   ```

2. **约束显式化**：禁止相邻时隙速度差过大
   ```
   |velocity[t] - velocity[t-1]| <= max_accel * dt
   ```

3. **初值条件**：设定合理的初始速度范围

### 根本问题
**L2b 轨迹优化器缺乏速度连续性约束**
- 优化器过度追求通信/能耗指标
- 生成不现实的"瞬移"轨迹
- BCD 循环因此失败回退到 L1（fallback 模式）
