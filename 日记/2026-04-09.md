# 2026-04-09 工作日记

## 主要工作

### 1. SINR 信道模型（precompute.py）

将上行链路速率计算从 SNR 改为 SINR（多用户干扰模型）：
- 新增 Step 0：预计算所有活跃任务的信道增益 `_sinr_gains`，构造最坏情形干扰总量 `_total_interf`
- 干扰 `I_{i,j,t} = Σ_k P_i * g_{k,j,t} - P_i * g_{i,j,t}`（所有活跃任务总干扰减去自身）
- 新增 `_rate_from_gain_sinr()` 函数：`r = B * log1p(P*g/(N0+I)) / ln2`
- 效果验证：seed=42 场景，max 并发 19 任务，I = 13x N_0，r_up 从 11.18 Mbps 降至 1.16 Mbps（降 86%），D_hat_offload 从 0.031 升至 0.302（增 677%）

### 2. evaluator.py 权重拆分

将 `energy_weight` 拆分为独立的 `energy_weight`（计算能耗γ）和 `prop_weight`（飞行能耗λ）。

### 3. BCD/SCA 容错改进

- `bcd_loop.py`：L2b 轨迹优化失败时 `break`（优雅降级）而非 `raise`
- `trajectory_opt.py`：
  - SCA iter > 0 失败时早停返回上一轮结果
  - 通信时延约束改为**外点罚函数形式**（引入 slack 变量），解决硬约束导致的 infeasible 问题

### 4. 评估权重 bug 修复（run_all_experiments.py）

D1 的 `evaluate_solution` 之前使用默认权重 (1,1,1)，A 组使用 (alpha=350, gamma_w=0.01)，导致 A=58.84 vs D1=0.21 完全无法比较。现在两组均传入 config 中的 alpha/gamma_w/lambda_w。

修复后对比：A=58.84，D1=75.99（BCD 有非确定性，58-76 范围内波动）。

### 5. HS 评估权重透传（hsIndividualEdgeUav.py）

`evaluate_solution` 的三处调用均传入 `alpha/gamma_w/lambda_w`，与 D1 对齐。

### 6. Prompt 增强（base_prompt.py）

- 新增活跃时间窗统计（min/max/mean slots，tight-window 比例）
- 规则 16 加数值安全范围（exp 上限 3.0，系数范围 [0.01, 100]）
- 新增规则 17：时间窗紧迫度（window_tightness = T/window_size 作为系数）

### 7. 参数调整（setting.cfg）

- `gamma_w`: 1.0 → 0.01（压低计算能耗项权重）

## 遗留问题

1. **BCD 2 轮收敛**：cost_history [4.83, 4.83]，立即触发 eps_bcd=1e-3 阈值，鞍点结构性局限
2. **种群多样性崩溃**：A 组 Gen2 起所有个体得分完全一致（58.84），HS 不再进化
