### 步骤4：核心生成逻辑

填充 5 个 `NotImplementedError` 方法：

| 方法 | 逻辑 |
|---|---|
| `_generate_time_slots` | `list(range(T))` |
| `_generate_active_slots` | contiguous_window 模式，`rng.integers` 采样窗口长度+起始 |
| `_generate_tasks` | 循环 numTasks，uniform 采样 pos/D_l/D_r/F/tau，调用 active_slots |
| `_generate_uavs` | 循环 numUAVs，固定 depot，N_max=None |
| `_build_meta` | 打包 T/delta/x_max/y_max/H/depot_pos/active_mode |

关键决策（三方协同确认）：
- 全部 uniform 采样（v1 不引入 log-uniform）
- 所有数值显式 `float()`/`int()` 转换，防止 numpy 标量污染 JSON
- `rng.integers(low, high+1)` 写法处理闭区间
- Codex review: 9/10

### 步骤5：校验逻辑

实现 `_validate_config`（前置）和 `_validate_scenario`（后置）：

**前置校验**（~55行）：
- 正值检查：T, delta, x_max, y_max, H, numTasks, numUAVs, E_max
- 范围对检查：D_l, D_r, F, tau, active_window
- 下界正值：D_l_min, D_r_min, F_min, tau_min
- 活跃窗口约束、active_mode 支持集、depot 边界
- 通信/硬件参数正值：B_up, B_down, P_i, P_j, N_0, rho_0, f_max, f_local_default
- 累积 errors 列表，一次性 raise ValueError

**后置校验**（~80行）：
- 硬约束：time_slots 一致性、数量一致性、索引唯一且匹配、位置边界、active 非空/范围/连续性/窗口长度、UAV depot 固定
- 软约束：tau 可行性粗估（本地+卸载都超时才算不可行），>80% 发 RuntimeWarning

**辅助方法**：
- `_is_within_map`：位置边界检查
- `_is_contiguous`：时隙连续性检查
- `_estimate_offload_delay_lb`：乐观卸载时延下界（实际距离计算）

Codex review: 8/10
