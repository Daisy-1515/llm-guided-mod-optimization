### 下午：① 数据结构 `dataCommon.py`

**目标**：新增 `ComputeTask` 和 `UAV` 两个数据类。

**三方协作过程**：
- Gemini 建议用 `@dataclass` + 预埋 `assigned_uav`/`trajectory` 等运行时字段
- Codex 建议用普通 class，不加未使用字段
- 最终决策：**普通 class**（与原项目一致）+ **只描述实体固有属性**（不存储算法中间结果）

**关键设计决策**：
1. 普通 class 而非 dataclass — 避免同一文件风格割裂
2. 不加 assigned_uav/trajectory/remaining_energy — 数据类是输入描述，不是状态容器
3. N_max 始终设为属性（默认 None） — 避免属性存在性不确定
4. f_local 放 ComputeTask — 公式 $f_i^t$ 下标带 $i$，支持异构设备
5. `active` 用 `defaultdict(bool)` — Codex review 发现未填充时隙会 KeyError，修复后安全返回 False

**提交**：`70d93c0`

### 下午（续）：①.5 Config 适配

**目标**：在 `setting.cfg` 和 `config.py` 中新增 Edge UAV 物理参数。

**参数来源**：底层变量清单.md（1. 物理环境 + 3. 设备硬件 + 5. 优化权重）
