### 上午（续）：② 场景生成器设计方案

与 Codex + Gemini 三方讨论，产出 `文档/场景生成器设计方案.md`（258 行），提交 `ca89343`。

**关键设计决策**：

1. **固定基站方案** — 所有 UAV 从同一 depot 出发/返回，depot_pos 默认地图中心，可配置
2. **连续窗口 active[t]** — 每个设备在随机连续时间窗口内活跃（非全时段），用 `defaultdict(bool)` 实现
3. **EdgeUavScenario dataclass** — 作为场景返回值，包含 tasks/uavs/time_slots/seed/meta，不含预计算量
4. **新建文件并行共存** — `edgeUavScenarioGenerator.py` 与原 `scenarioGenerator.py` 并行，不修改旧 HS 链路
5. **3 个新 config 节** — edgeUavTask（12 参数）、edgeUavDepot（2 参数）、edgeUavSeed（1 参数）
6. **场景可行性校验** — 活跃时隙/索引唯一/位置边界/tau 粗估
7. **序列化** — save_json / load_json 支持固定场景横向对比

**8 步实施计划**：冻结接口 → 扩展 config → 搭骨架 → 核心生成 → 校验 → 序列化 → Smoke Test → Review
