| 决策 | 结论 | 理由 |

| 向后兼容 | 默认 `individual_type="multi_call"` | 原 Taxi-Passenger 链路零修改 |

### LLM API 改造

#### 问题

`llmInterface_huggingface.py` 的 `getResponse()` 只处理 DeepSeek-R1 的 `<think>` 标签格式。无标签时返回 `None`，导致 GPT / GLM / Qwen 等模型无法使用。

#### 改动

**`llmAPI/llmInterface_huggingface.py`**（3 处）：

1. **`_normalize_endpoint()`** — endpoint 自动补全 `/chat/completions`
   - 用户填 `/v1` 或完整路径均可

2. **`_parse_content()`** — 三路径响应解析：
   - 路径 1：`<think>...</think>` → 提取 `</think>` 后 JSON（DeepSeek-R1）
   - 路径 2：无标签 → 直接从 content 提取 JSON（GPT/GLM/通用）
   - 路径 3：兜底 → 返回原始文本

3. **`getResponse()`** — 简化，调用 `_parse_content()` 统一处理

**`config/setting.cfg`**：
```ini
[llmSettings]
platform = HuggingFace
model = glm-5
```

#### 验证

- 3 条解析路径单元测试 ✅
- deepseek-reasoner 真实 API 调用 → `{"status": "ok"}` ✅
- glm-5 真实 API 调用 → `{"status": "ok"}` ✅
